SECTION 01
アプリの概要 ・ 起動方法
Overview & Getting Started
◆

TimelySchool 受付アプリは、STRAWBERRY FIELD Children's Garden の一時預かり・自園生徒の早朝延長サービスにおける、受付からお会計、領収書発行までを一括で行うアプリです。

対応デバイス
DeviceデバイスUse主な用途
iPadお客様への入力依頼、スタッフの受付業務、お会計
デスクトップPC顧客管理、日報・月報の確認、データ管理
iPhone / Android緊急時の参照用
アプリの起動
1
ファイルをダブルクリック
配布された TimelySchool受付_v36.html ファイルをダブルクリックで開きます。Chrome または Safari が起動します。
2
ブックマークまたはホーム画面に追加
毎回開きやすいように、ブラウザのブックマーク (PC) またはホーム画面 (iPad: Safariの共有 → 「ホーム画面に追加」) に登録しておきましょう。
◆ NOTE
現在 (v36) のアプリは、そのブラウザだけにデータを保存します。同じiPadで使い続ければデータは残りますが、別の端末からは見えません。クラウド化 (Firebase化) は次のフェーズで実装予定です。
SECTION 02
ホーム画面の見方
Home Screen
◆

アプリを開くと、まずホーム画面が表示されます。中央に 初めてのご利用 の大きなボタン、下に小さく「Staff Only」のメニューバーがあります。

画面の構成
Element要素Role役割Who対象
初めてのご利用 新規のお客様 / 本人確認 お客様
予約リスト 事前予約の登録・確認 スタッフ
お会計待ち お会計待ちのお客様一覧 スタッフ
自園生徒 早朝・延長の自園生徒利用記録 スタッフ
管理メニュー 顧客管理・名簿・日報月報・データ 事務
◆ TIP
「初めてのご利用」ボタンは大きく、Staff Onlyのバーは小さくデザインされています。これは、iPadをお客様に渡した時、迷わないようにするためです。
SECTION 03
お客様の受付フロー
Customer Reception Flow
◆

お客様がご来園された際の標準フローは以下の通りです。

1
iPadをお客様にお渡しする
「ホーム画面」の状態で iPad をお客様にお渡しします。
2
お客様が「初めてのご利用」をタップ
初回 → 新規登録画面 / 2回目以降 → 本人確認画面 にお客様自身で進んでいただきます。
3
お客様が必要事項を入力
お子様情報、保護者情報、当日のお預け情報を入力していただきます。
4
同意 → スタッフ確認画面
お客様が同意ボタンを押すと「お礼画面」になります。iPadをスタッフが受け取り、内容を確認します。
5
お預かり開始
スタッフが「保存」ボタンを押し、お預かり開始です。データは「お会計待ち」リストに入ります。
6
お迎え時 ・ お会計
お迎え時に「お会計待ちリスト」から該当のお客様を選び、お会計 → 領収書発行。
SECTION 04
新規のお客様 ・ 初めてのご利用
New Guest Registration
◆

初めてご利用のお客様は、お子様の基本情報 + 保護者情報 + 同意・署名を登録していただきます。

入力項目
Section区分Items項目
お子様情報 氏名 / フリガナ / 性別 / 生年月日 / 血液型 / 平熱 / かかりつけ医
アレルギー 14項目から選択 (卵・乳・小麦・大豆 など) + その他フリーテキスト
保護者情報 氏名 / フリガナ / 電話番号 / ご住所
緊急連絡先 緊急時連絡先のお名前 / 電話番号
同意 ・ 署名 ご利用条件の同意チェック + 署名
⚠ IMPORTANT
姓と名の間は必ずスペースを入れてください (例: 高嶋 仁)。スペースがないと領収書の宛名が正しく出ません。
◆ TIP
登録後、自動的に「来園登録」画面に遷移します。同じ流れで当日のお預け情報を入力していただきます。
SECTION 05
2回目以降のお客様 ・ 本人確認
Verify Identity
◆

2回目以降のお客様は、本人確認を行い、登録済みの情報をすぐに呼び出します。

本人確認の手順
1
お子様の生年月日を入力
年 / 月 / 日 の3つを選択します。
2
保護者の電話番号下4桁を入力
登録時の電話番号の最後の4桁を入力します。
3
一致すれば来園登録画面へ
該当するお子様の情報が表示され、当日のお預け情報入力に進めます。
◆ NOTE
生年月日と電話下4桁が一致しない場合は、同じ電話番号で複数のお子様が登録されている可能性があります。スタッフがホーム画面 → 管理メニュー → 顧客一覧で確認してください。
SECTION 06
来園登録 ・ 当日入力
Visit Registration
◆

当日のお預け情報を入力する画面です。4つのセクションに分かれています。

A・ご利用日 ・ 時刻

大きなカードでご利用日を表示。来園時刻 + お迎え予定時刻を時刻ピッカーで選択します。

B・本日の体調 ・ ミルク (乳児向け)

体温、ミルク利用の有無、最後にミルクを飲んだ時刻・次回予定時刻などを入力します。ミルクを飲まないお子様は「飲んでいない」を選択してください。

C・持ち物

当日お持ちいただいた持ち物 (オムツ、着替え、スタイ等) にチェック。お会計時に「返却済」を確認するため必須です。

D・特記事項 ・ 同意

特記事項のフリーテキスト + 利用条件の同意チェック。

→
同意 → Thank You 画面 → スタッフ確認画面 → 保存
お客様が同意 → 「ご入力ありがとうございました」のお礼画面。iPad をスタッフが受け取り、内容のサマリーを確認。問題なければ「保存」をタップ。
SECTION 07
お会計 ・ 領収書発行
Checkout & Receipt
◆

お迎え時にお会計を行います。ホーム → お会計待ち から該当のお客様を選択します。

お会計画面の流れ
1
実際のお迎え時刻を入力
画面中央の「Actual Pickup ・ 実お迎え」を時刻ピッカーで設定します。
2
料金が自動計算される
基本料金 + 延長料金が自動表示されます。お客様にも見せて確認します。
3
お伝え事項を入力
ミルク (時刻・量)、便 (時刻・状態)、午睡 (時刻)、持ち物返却。「なかった」場合もチェック必須。
4
受領者を選択
プルダウンからお会計を受け取ったスタッフを選択。
5
「お会計を承認 → 領収書発行」をタップ
領収書画面が表示されます。印刷 / PDF保存 でA4印刷またはPDF保存。
6
「会計完了 → ホームへ戻る」
会計完了です。お疲れさまでした。
⚠ IMPORTANT
領収書印刷時は、Chrome の印刷ダイアログで 「背景のグラフィック」を ON にしてください。OFFのままだとゴールドの罫線などが印刷されません。
◆ NOTE
「持ち物返却」は領収書に必ず表示されます。「返却済」「未返却 (赤色で警告)」「持ち込みなし」のいずれかが領収書に印字されます。
SECTION 08
自園生徒の利用記録
Internal Students
◆

自園生徒の早朝(¥700/日) ・ 延長(¥700/日)利用を記録する画面です。スタッフ専用

2つの入力方式 (ハイブリッド型)

① 自園生徒名簿から選択: 事前に登録した生徒からタップで選択。クラス (Leaf / Blossom / Harvest) や五十音で絞り込み可能。

② 手入力: 名簿にない子は「↓ 名簿にない子を手入力する」リンクで手入力できます。「この子を自園生徒名簿に登録する」チェックを入れると同時に名簿登録されます。

入力手順
1
利用区分を選択
「自園生徒・早朝」または「自園生徒・延長」を選択。
2
生徒を選ぶ (名簿 or 手入力)
クラスタブ・五十音タブで絞り込み → カードをタップ。
3
ご利用日 ・ 時刻を入力
日付 / 開始時刻 / 終了予定時刻。
4
「記録を保存 (会計待ちへ)」をタップ
お会計待ちリストに入ります。お迎え時にお会計してください。
◆ TIP
自園生徒のお会計画面では「お伝え事項」は表示されません (日々の連絡は別途行っているため)。実お迎え時刻と受領者だけで完了します。
SECTION 09
予約リストの活用
Reservations
◆

事前にご予約のあるお客様を予約リストに追加しておくと、当日の受付がスムーズになります。

予約追加の流れ
1
ホーム → 予約リスト
「予約リスト」をタップして画面を開く。
2
お客様を検索
名前または五十音 (カタカナ) で検索。例: 「タカハシ」「サ行」
3
「予約に追加」 → 日付 ・ 時刻を入力
予約日とご来園予定時刻を選択。
当日の運用

ご来園時、予約リストから該当のお客様の「当日入力を開始」をタップ → iPadをお客様にお渡し。すでに顧客情報は紐づいているので、当日入力だけを行えばOKです。

SECTION 10
管理メニュー
Administration
◆

顧客情報、自園生徒名簿、日報・月報、データ管理など。事務担当・園長

3つのタブ
TabタブContent内容
顧客一覧 登録されたお客様の一覧。カードをクリックで編集可能。名前検索 + 五十音索引で絞り込み。
来園記録一覧 過去の来園記録の一覧。月別・未会計優先などで並び替え可能。
自園生徒名簿 自園生徒のマスター名簿。「自園生徒を追加」ボタンで登録。クラス・五十音で絞り込み。
日報 ・ 月報

「日報・売上管理」ボタンから。前日/翌日のナビゲーション、日別・月別の売上集計、CSV出力に対応。

データ管理

顧客リストをコピー (Google Sheets貼り付け用) / JSON出力 / JSON読込 / 全データ削除 が可能です。

⚠ DANGER
「全データ削除」は元に戻せません。誤って押さないよう注意してください。テスト期間中はこの操作はしないでください。
SECTION 11
毎日の終業作業 ・ データバックアップ
Daily Backup
◆

必ず毎日終業時にデータバックアップを取ってください。現在のアプリはブラウザ内にデータを保存しているため、誤ってキャッシュクリアすると全データが消失します。

バックアップ手順
1
管理メニューを開く
ホーム → Staff Only バーの「管理メニュー」をタップ。
2
データ管理 → 「データをJSON出力」
画面下部の「データ管理」セクションで実行。
3
ファイルを保存 (日付を含める)
例: timelyschool_backup_2026-05-22.json など。
4
Google ドライブ等にアップロード
事務PCのGoogleドライブの専用フォルダに保管。最低7日分は残してください。
⚠ CRITICAL
このバックアップを怠ると、iPad/PCのブラウザの不具合やキャッシュクリアで全顧客データが消失します。必ず毎日実施してください。クラウド化 (Firebase化) 完了後はこの作業は不要になります。
SECTION 12
よくあるご質問 ・ トラブル対応
FAQ & Troubleshooting
◆
Q.お客様が本人確認画面で「該当するお客様が見つかりません」と出る
①生年月日・電話番号下4桁が間違っていないか確認。 ②管理メニュー → 顧客一覧でお客様を検索 → 登録があるか確認。 ③別の電話番号で登録されている可能性があります。スタッフが直接来園登録画面に飛ばすことも可能。
Q.領収書がA4に収まらない / レイアウトが崩れる
印刷時に Chrome の印刷ダイアログで「背景のグラフィック」を ON にしてください。デフォルトでは65%スケールで印刷されるよう設定されています。
Q.誤ってお会計を完了してしまった (修正したい)
管理メニュー → 来園記録一覧から該当の記録を探し、「修正」ボタンから再度お会計画面に入れます。実お迎え時刻・受領者なども修正可能。
Q.iPadとPCでデータが違って見える
現在 (v36) は端末ごとにデータが独立しています。データを共有するには JSON 出力・読込で手動同期するか、クラウド化 (Firebase) を待つ必要があります。
Q.自園生徒の名前が五十音検索に出てこない
登録時にフリガナを入力していない可能性があります。管理メニュー → 自園生徒名簿で該当生徒を編集し、フリガナを追加してください。
Q.同意画面でお客様が誤って「キャンセル」した
入力情報はクリアされません。「初めてのご利用」または「予約リスト」から再開できます。
Q.ブラウザが固まった / 動かない
ブラウザのタブを一度閉じて、再度開いてください。データは保存されているので、入力途中のものでなければ問題ありません。
SECTION 13
フィードバックの送り方
Feedback
◆

v36 はテスト運用版です。使いにくさ・不具合・要望は、すべて記録して開発担当者に共有してください。良くなるかどうかは、皆さまのフィードバックにかかっています。

記録すべきこと
  • 発生日時 (例: 2026/5/22 14:30)
  • 操作者 (お客様 / スタッフ名)
  • どの画面で起きたか (例: 来園登録画面の持ち物セクション)
  • 何をしたら何が起きたか (例: 「保存」ボタンを押したら画面が真っ白になった)
  • 期待していた動作
  • (可能なら) スクリーンショット
